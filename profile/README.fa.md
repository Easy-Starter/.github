# Easy Starter

**تمپلیت‌های Production-capable و AI-first برای فریلنسرها، solopreneurها و تیم‌های محصول.**

از یک پایه‌ی یکپارچه شروع کنید، اول Spec بنویسید و زمان پروژه را به‌جای تنظیمات تکراری روی فیچرهای محصول بگذارید.

## چرا Easy Starter؟

ایجنت‌های کدنویسی می‌توانند سریع کد تولید کنند؛ اما سرعت بدون معماری مشترک، پروژه‌های ناسازگار و سخت‌نگهداری می‌سازد. Easy Starter پایه‌های Opinionated، گیت‌های کیفیت تکرارپذیر، قوانین قابل‌خواندن برای ایجنت و Specهای عملی محصول فراهم می‌کند.

## اصول اصلی

- **Spec قبل از کد:** فیچرهای مهم با نیازمندی و معیار پذیرش شروع می‌شوند.
- **مستقل از ایجنت:** Cursor، Codex، Claude، Copilot و ابزارهای آینده از قوانین یکسان ریپو پیروی می‌کنند.
- **Production-capable:** تنظیمات امن، تست، CI، اعتبارسنجی محیط و پروفایل استقرار جزو پایه هستند.
- **هسته‌ی کوچک و ماژول‌های اختیاری:** استارترها Dependency و فرض محصولی غیرضروری تحمیل نمی‌کنند.
- **دوزبانه از ابتدا:** مستندات اصلی محصول و پروژه می‌توانند انگلیسی و فارسی باشند.

## وضعیت پروژه

Easy Starter به‌صورت عمومی در حال توسعه است. ریپوهای دارای وضعیت **foundation** هنوز برای استفاده‌ی پایدار در پروداکشن آماده نیستند. نسخه‌ی پایدار از `v1.0.0` شروع می‌شود.
