# مشارکت در Easy Starter

از مشارکت شما ممنونیم. Easy Starter بر تغییرات متمرکز، Spec شفاف، معماری قابل‌پیش‌بینی و مستندات عملی تأکید دارد.

## پیش از شروع

- Issueها و Discussionهای موجود را جست‌وجو کنید.
- برای تغییر بزرگ، Breaking، چندریپویی یا معماری ابتدا Issue باز کنید.
- فایل `AGENTS.md`، Spec مرتبط و قواعد محلی ریپو را بخوانید.
- Credential، داده‌ی خصوصی مشتری یا کد مالکیتی را وارد نکنید.

## ورک‌فلوی توسعه

۱. از شاخه‌ی پیش‌فرض فعلی Fork یا Branch بگیرید.
۲. هر Branch و Pull Request را به یک موضوع محدود کنید.
۳. تست، مستندات، مثال و Migration Note لازم را اضافه یا به‌روزرسانی کنید.
۴. تمام Checkهای ریپو را محلی اجرا کنید.
۵. با قالب پیش‌فرض Pull Request باز کنید.
۶. بازخوردها را اعمال و Branch را به‌روز نگه دارید.

## الزامات تغییر

- Dependency جدید به دلیل مستند و ارزیابی نگهداری نیاز دارد.
- تغییر معماری باید ADR در `docs/decisions/` داشته باشد.
- تغییر قابل‌مشاهده برای کاربر نیازمند مستندات است.
- Breaking Change به Migration Note و تغییر نسخه‌ی مناسب نیاز دارد.
- مسئولیت کد تولیدشده یا کمک‌گرفته از AI همچنان با مشارکت‌کننده است.

با مشارکت، Code of Conduct را می‌پذیرید و Contribution خود را تحت License همان ریپو ارائه می‌کنید.
