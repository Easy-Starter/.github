# سیاست امنیتی

[English Version](../SECURITY.md)

## نسخه‌های پشتیبانی‌شده

اصلاحات امنیتی برای آخرین Major Version پایدار هر ریپو ارائه می‌شود. ریپوهای دارای وضعیت `foundation`، نسخه‌های قبل از `1.0.0`، ریپوهای Archiveشده یا موارد صراحتاً پشتیبانی‌نشده تعهد پشتیبانی پروداکشن ندارند.

## گزارش آسیب‌پذیری

Issue عمومی باز نکنید. از گزینه‌ی **Report a vulnerability** در تب Security ریپو استفاده کنید. اگر Private Vulnerability Reporting فعال نیست، از طریق پروفایل سازمان به‌صورت خصوصی با Maintainerها تماس بگیرید.

موارد زیر را ارائه کنید:

- ریپو و نسخه یا Commit آسیب‌پذیر
- اثر و سناریوی واقعی حمله
- مراحل بازتولید یا Proof of Concept حداقلی
- راهکار کاهش ریسک یا Workaround شناخته‌شده
- وضعیت افشای قبلی مسئله

پیش از افشای عمومی، زمان کافی برای بررسی بدهید. Credential واقعی، داده‌ی مشتری یا Payload مخرب در گزارش قرار ندهید.
