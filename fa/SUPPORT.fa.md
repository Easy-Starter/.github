# پشتیبانی

[English Version](../SUPPORT.md)

## کجا سؤال بپرسیم؟

- **سؤال راه‌اندازی یا استفاده:** Discussion ریپوی [`easy-starter-docs`](https://github.com/easy-starter/easy-starter-docs/discussions).
- **باگ قابل‌بازتولید:** فرم Bug Report در ریپوی مرتبط.
- **پیشنهاد قابلیت:** فرم Feature Request در ریپوی مرتبط.
- **مشکل مستندات:** Documentation Issue در `easy-starter-docs`.
- **آسیب‌پذیری امنیتی:** طبق `SECURITY.md` و هرگز در Issue عمومی.

## قبل از ثبت درخواست

Issueهای موجود را جست‌وجو کنید، از آخرین نسخه‌ی پشتیبانی‌شده استفاده کنید، محیط و فرمان‌های دقیق را بنویسید، بازتولید حداقلی ارائه دهید و Credential یا داده‌ی خصوصی را حذف کنید.

پشتیبانی Community به‌صورت Best-effort است و تمپلیت‌های عمومی شامل SLA یا مشاوره‌ی اختصاصی پروژه نیستند.
