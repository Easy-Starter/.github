# Contributing to Easy Starter

[نسخه فارسی](./fa/CONTRIBUTING.fa.md)

Thank you for contributing. Easy Starter values focused changes, clear specifications, predictable architecture, and practical documentation.

## Before you start

- Search existing issues and discussions.
- Open an issue before large, breaking, cross-repository, or architectural changes.
- Read the repository `AGENTS.md`, relevant specification, and local contribution notes.
- Never include credentials, private customer data, or proprietary code.

## Development workflow

1. Fork or branch from the current default branch.
2. Keep one concern per branch and pull request.
3. Add or update tests, documentation, examples, and migration notes.
4. Run the complete repository check locally.
5. Open a pull request using the default template.
6. Address review comments and keep the branch current.

## Change requirements

- New dependencies require a documented reason and maintenance assessment.
- Architecture changes require an ADR under `docs/decisions/`.
- User-facing changes require documentation.
- Breaking changes require migration notes and an appropriate version change.
- Generated or AI-assisted code remains the contributor's responsibility.

## Commit and pull request quality

Use clear, imperative commit messages. Pull requests should explain the problem, approach, verification, risks, and related specification or issue.

By contributing, you agree to follow the Code of Conduct and license your contribution under the repository's license.
